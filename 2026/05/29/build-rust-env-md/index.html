在VS code中搭建Rust开发环境

打开 VS Code，直接在扩展市场（Marketplace）中搜索安装：Rust-Analyzer。

对于代码调试，我的做法是直接在终端用 GDB 来调试 Rust 编译出来的二进制文件。因为 Rust 和 C 语言一样，都是直接编译成原生机器码的系统级语言，并且同样使用 DWARF格式来存储调试符号。虽然普通的 gdb 就能直接加载 Rust 程序，但由于 Rust 有很多独有的高级类型（比如 String、Vec、Option 以及带有变体的 Enum），原生的 GDB 在打印（print）这些变量时，只会把它当作一堆凌乱的 C 结构体和裸指针打印出来，非常痛苦。为了解决这个问题，Rust 官方提供了一个名为 rust-gdb 的包装脚本。它本质上就是调用你系统里的 gdb，但自动加载了一套 Python 脚本（Pretty-printers）。这套脚本能把 Rust 独有的复杂数据结构，在终端里还原成人类可读的 Rust 语法格式。开发者不需要单独安装它，通过 rustup 安装好环境后，它已经躺在你的终端里了：

使用rust-gdb调试手法和gdb几乎一模一样。
